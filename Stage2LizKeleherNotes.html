Work Session 2 Notes

Variables

  • Allows us to keep track of values and use names to refer to them
  • Assigning values to variables gives a specific one a specific value throughout the rest of your code.
  • There is a difference between = in programming and = in math. In programming, = means given the value of
  • It's crucial to give values to all of your variables because others who look at your code will need it and computers are stupid.

Strings

  • Strings are a sequence of characters surrounded by single or double quotes.
  • If you want the words to show up, you have to put them in quotes.
  • If you use words without them, you must first define them as a variable.
  • + means "concatenate" or put together (not add)
  • put spaces inside your quotes to indicate where the space should be
  • indexing: use brackets and numbers to choose one thing from the string (positions in strings start with 0) You can also use negative numbers if you don't know where the end is, or it is a long string
  • sub-sequencing: choose a rnage of things by using a semicolon [expression: expression] If you want the beginning to the number or the number to the end, leave a space before or after the brackets
  • finding strings in strings: .find is a procedure in Python. The output is the position of the first occurence. Capitalization matters. If the string can't be found, the output will be -1. no quotes is a variable you need to define. “” or ‘’ is exactly what you type so, s.find(s) is always 0 and s.find(’s’) depends on if there is an actual letter s in the variable s
  • an empty string exists before and between all things in a string. These spaces count as a position.
  • Parameter: In "text".find('t') the 't' is the parameter.
  • If you count from the beginning of a string, when choosing the end, choose the number after because it is inclusive. Going from the end backward is exclusive, so choose the last position you want.
  • Use the shortest possible solution to allow you to apply it to many things

  • Else: If the first statement is false, indicate what the computer should do with else.

  • OR only evaluates when you need it (i.e. if it finds the first thing to be true, but you have an undefined variable on the other side, it will stop and just return true.
